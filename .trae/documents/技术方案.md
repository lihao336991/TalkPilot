## 三、模块一：实时对话引擎（客户端直连 Deepgram）

### 3.1 核心设计：客户端直连

**关键决策：ASR 不走后端，客户端直接 WebSocket 连接 Deepgram。**
好处：

- 省掉整个 WebSocket Gateway 服务端
- 延迟更低（少一跳）
- 后端完全 Serverless 化
  安全性：
- Deepgram API Key 不能暴露给客户端
- 解决方案：通过 Supabase Edge Function 生成短期 token

```typescript
// Edge Function: /functions/v1/deepgram-token
// 生成 Deepgram 临时 API Key（有效期 10 分钟）
import { serve } from  https://deno.land/std@0.168.0/http/server.ts ;
import { createClient } from  https://esm.sh/@supabase/supabase-js@2 ;

serve(async (req) => {
  // 1. 验证用户身份
  const supabase = createClient(
    Deno.env.get( SUPABASE_URL )!,
    Deno.env.get( SUPABASE_ANON_KEY )!,
    { global: { headers: { Authorization: req.headers.get( Authorization )! } } }
  );
  const { data: { user }, error } = await supabase.auth.getUser();
  if (error || !user) return new Response( Unauthorized , { status: 401 });

  // 2. 检查用量限制（Free 用户每天 15 分钟）
  const usage = await checkDailyUsage(supabase, user.id);
  if (usage.exceeded) {
    return Response.json({ error:  Daily limit exceeded  }, { status: 429 });
  }

  // 3. 向 Deepgram 申请临时 Key
  const resp = await fetch( https://api.deepgram.com/v1/keys/  + Deno.env.get( DG_PROJECT_ID ), {
    method:  POST ,
    headers: {
       Authorization :  Token   + Deno.env.get( DEEPGRAM_API_KEY ),
       Content-Type :  application/json ,
    },
    body: JSON.stringify({
      comment: `temp-key-${user.id}`,
      scopes: [ usage:write ],
      time_to_live_in_seconds: 600, // 10 分钟有效
    }),
  });

  const { key } = await resp.json();
  return Response.json({ deepgram_token: key, expires_in: 600 });
});
```

### 3.2 客户端音频采集与 ASR

**当前实现对齐**：

- 音频采集使用 `react-native-live-audio-stream` 输出 16k / mono / PCM chunk，直接发送给 Deepgram WebSocket。
- iOS 侧通过 `expo-av` 设置录音优先 audio mode，并通过本地 Expo plugin 注入 `VoiceChatModule`，在录音开始/结束时动态启用/关闭 `AVAudioSession.playAndRecord + voiceChat`。
- `DeepgramStreamingService` 不再在 `Results.is_final=true` 时立即提交 turn，而是：
  - 在同一轮 utterance 内累计多个 final 片段
  - 等 `UtteranceEnd` 到来时一次性提交 turn、写库、再触发 suggest/review
- 这样可以减少短暂停顿导致的“提前切句”和“同一轮多段 final 只保留最后一段”的问题。

```typescript
// 当前主链路伪代码（以仓库实现为准）
await audioEngine.init(); // expo-av audio mode + iOS voiceChat 预热
const token = await deepgramTokenService.getToken();
await deepgramService.connect(token, handleUtteranceEnd);
await audioEngine.start((base64Chunk) => {
  deepgramService.sendAudio(base64Chunk);
});

// Deepgram Results
if (data.type === "Results") {
  const transcript = data.channel?.alternatives[0]?.transcript ?? "";
  const words = data.channel?.alternatives[0]?.words ?? [];
  const speaker = determineSpeaker(words);

  if (data.is_final) {
    // 同一轮 utterance 内累计多个 final 片段
    currentTurn.text = mergeTranscriptSegments(currentTurn.text, transcript);
  } else {
    // interim 仅作为 UI 展示
    updateInterim(transcript, speaker);
  }
}

// Deepgram UtteranceEnd
if (data.type === "UtteranceEnd") {
  // 快照当前 turn，避免异步回调覆盖
  const committedTurn = snapshotCurrentTurn();
  clearInterimAndMutableState();
  await sessionManager.recordTurn(committedTurn);
  await handleUtteranceEnd(committedTurn); // self -> review, other -> suggest
}
```

### 3.3 说话人分离方案（轻量版）

**当前实现口径：Deepgram 内置 Diarization + 轻量 self/other 映射，不是真正声纹识别**

```plaintext
流程：
1. 开启 Deepgram 的 diarize=true 参数
2. Deepgram 自动为每个 word 标注 speaker: 0 或 speaker: 1
3. 客户端对当前 turn 的 words.speaker 做多数派统计
4. 将该 speaker id 与本地 selfSpeakerId 比较
5. 相等记为 self，否则记为 other
```

注意：

- 这不是生物特征层面的 voiceprint / speaker verification。
- 当前代码里的 speaker calibration 仍偏 MVP 级，准确率依赖 Deepgram diarization 稳定性。
- 如果后续要显著提升准确率，需要补“动态标定”或真正的声纹服务。

| 方案                           | 实现成本           | 准确率 | 额外费用     |
| ------------------------------ | ------------------ | ------ | ------------ |
| Deepgram diarize（MVP 选这个） | 零代码             | ~85%   | +$0.0013/min |
| 耳机双通道                     | 需要 native module | ~95%   | 无           |
| 自建声纹                       | 需要部署模型       | ~88%   | 服务器成本   |

MVP 先用 Deepgram 内置 diarize（加一个参数就搞定），后续体验不够好再升级到耳机双通道方案。

### 3.4 端侧 ASR 方案：Whisper.cpp（Free 用户零成本）

#### 3.4.1 方案概述

**核心思路：Free 用户使用端侧 Whisper 模型进行语音识别，ASR 成本直接归零。Pro 用户使用云端 Groq Whisper API 获得更高质量体验。**
这是降低 ASR 成本最关键的架构决策——ASR 占总变动成本的 53%，端侧化后 Free 用户的 ASR 成本从每月数千美元降为 $0。

| -------- | ------------------------ | ---------- | ------ | --------- | ---------------------------------- |
| **Free** | Whisper.cpp base.en 端侧 | **$0**     | ~12%   | ~400ms/段 | Deepgram diarize（仅标定阶段使用） |
| **Pro**  | Groq Whisper API 云端    | $0.011/min | ~8-10% | ~200ms/段 | Deepgram diarize                   |

#### 3.4.3 Whisper.cpp 端侧集成

**模型选择**：

| 模型        | 大小      | iPhone 12+ 延迟 | WER      | 推荐         |
| ----------- | --------- | --------------- | -------- | ------------ |
| tiny.en     | 75MB      | ~200ms/段       | ~15%     | 低端设备降级 |
| **base.en** | **142MB** | **~400ms/段**   | **~12%** | **MVP 首选** |
| small.en    | 466MB     | ~1.2s/段        | ~9%      | 后续可选升级 |

**React Native 集成（使用 whisper.rn 库）**：

```typescript
// WhisperEngine.ts — 端侧 ASR 引擎
import { initWhisper, WhisperContext } from "whisper.rn";

class WhisperEngine {
  private whisperCtx: WhisperContext | null = null;
  private modelReady = false;

  /**
   * 初始化 Whisper 模型
   * 首次使用时下载模型文件（142MB），后续从本地缓存加载
   */
  async initialize() {
    const modelPath = await this.ensureModelDownloaded("ggml-base.en.bin");

    this.whisperCtx = await initWhisper({
      filePath: modelPath,
      // 启用 CoreML 加速 (iOS) / NNAPI (Android)
      useCoreMLIos: true,
      useGpu: true,
    });

    this.modelReady = true;
  }

  /**
   * VAD 检测到一段语音结束后，调用端侧 Whisper 转录
   */
  async transcribe(audioFilePath: string): Promise<TranscribeResult> {
    if (!this.whisperCtx) throw new Error("Whisper not initialized");

    const startTime = Date.now();

    const result = await this.whisperCtx.transcribe(audioFilePath, {
      language: "en",
      maxLen: 0, // 不限段长
      translate: false,
      beamSize: 1, // beam=1 最快，质量损失小
      bestOf: 1,
    });

    const latency = Date.now() - startTime;
    console.log(`Whisper transcribe: ${latency}ms`);

    return {
      text: result.data
        .map((seg) => seg.text)
        .join(" ")
        .trim(),
      segments: result.data,
      latency_ms: latency,
    };
  }

  /**
   * 模型下载与缓存管理
   */
  private async ensureModelDownloaded(modelName: string): Promise<string> {
    const localPath = `${FileSystem.documentDirectory}models/${modelName}`;
    const exists = await FileSystem.getInfoAsync(localPath);

    if (!exists.exists) {
      // 首次下载，展示进度条
      const downloadUrl = `https://huggingface.co/ggerganov/whisper.cpp/resolve/main/${modelName}`;
      await FileSystem.downloadAsync(downloadUrl, localPath, {
        onProgress: (progress) => {
          useAppStore
            .getState()
            .setModelDownloadProgress(
              progress.totalBytesWritten / progress.totalBytesExpectedToWrite,
            );
        },
      });
    }

    return localPath;
  }
}
```

#### 3.4.4 VAD 切段 + Whisper 的完整流程

端侧 Whisper 不支持流式识别，需要配合 VAD（Voice Activity Detection）进行 说完一句就识别 ：

```plaintext
[持续录音] → [端侧 VAD 检测语音活动]
                ↓ 检测到停顿 > 1.5s
            [截取该段音频]
                ↓
            [送入 Whisper.cpp 端侧推理]
                ↓ ~400ms
            [返回文字结果]
                ↓
            [更新 UI + 触发建议/Review]

```

```typescript
// VADManager.ts — 端侧 VAD + Whisper 组合
import { useSileroVAD } from "@ricky0123/vad-react-native";

class VADManager {
  private whisperEngine: WhisperEngine;
  private audioRecorder: AudioRecorder;

  /**
   * 使用 Silero VAD 检测语音段落
   * Silero VAD 模型仅 ~2MB，端侧运行，零成本
   */
  startListening() {
    useSileroVAD({
      onSpeechStart: () => {
        // 开始录制当前语音段
        this.audioRecorder.startSegment();
      },
      onSpeechEnd: async (audioData: Float32Array) => {
        // 语音段结束，保存为文件
        const segmentPath = await this.audioRecorder.saveSegment(audioData);

        // 端侧 Whisper 转录
        const result = await this.whisperEngine.transcribe(segmentPath);

        if (result.text.trim().length > 0) {
          // 推送到 UI
          useConversationStore.getState().addTranscript({
            text: result.text,
            timestamp: Date.now(),
            source: "on-device",
          });
        }

        // 清理临时音频文件
        await FileSystem.deleteAsync(segmentPath);
      },
      // VAD 参数
      positiveSpeechThreshold: 0.8,
      negativeSpeechThreshold: 0.3,
      minSpeechFrames: 5,
      preSpeechPadFrames: 10,
    });
  }
}
```

#### 3.4.5 端侧方案的说话人分离

端侧 Whisper 本身不支持 diarize，需要额外方案：

| 方案                         | 说明                                                   | 推荐度   |
| ---------------------------- | ------------------------------------------------------ | -------- |
| **用户标定 + 音量/方向推断** | 利用麦克风方向和音量差异（自己说话声音更大）做粗略区分 | MVP 可用 |
| 耳机物理分离                 | 耳机麦克风=自己，手机麦克风=对方                       | 最准确   |
| 端侧声纹（sherpa-onnx）      | 轻量端侧声纹模型，~5MB                                 | 后续升级 |

MVP 阶段推荐最简方案：

```typescript
// 简易说话人区分：基于音量阈值
function classifySpeaker(audioData: Float32Array): "self" | "other" {
  const rms = Math.sqrt(
    audioData.reduce((sum, x) => sum + x * x, 0) / audioData.length,
  );
  // 自己的声音离麦克风近，RMS 通常更高
  // 阈值在首次标定时确定
  return rms > selfVolumeThreshold ? "self" : "other";
}
```

```typescript
      method: "POST",
      headers: {
        Authorization: `Bearer ${accessToken}`,
      },
      body: formData,
    });

    return response.json();
  }
}
```

## 四、模块二：AI 实时回复建议

### 4.1 触发流程（全部走 Edge Function）

```plaintext
对方说完 (UtteranceEnd)
    ↓
客户端发 HTTP POST 到 Edge Function
    ↓
Edge Function: 从 Supabase DB 读取最近上下文
    ↓
Edge Function: 调用统一 LLM Runtime
    ↓
Edge Function: 返回 JSON 建议结果
    ↓
客户端仅在结果返回后渲染建议气泡
```

**当前实现对齐**：

- `suggest` 已回退为普通 HTTP JSON 返回，不再走 SSE/ReadableStream。
- `review` 也是普通 JSON 返回。
- `suggest` / `review` 响应 header 会额外暴露：
  - `X-LLM-Provider`
  - `X-LLM-Model`
  - `X-LLM-Key-Prefix`
- 前端 debug 面板会读取这些 header，用于追踪当前实际命中的模型配置。

### 4.2 Edge Function：生成建议

**当前实现对齐**：

- 建议生成当前输出为单条 `simple` 风格 quick reply，而不是 2 条 formal/casual。
- 前端展示为更强气泡感的 quick reply，而非大卡片；无 loading 占位，结果到了才显示。
- 新的 suggest 请求开始，或本人一句话完成进入 review 流程时，旧 suggest 会立刻清空。
- 为避免竞态，前端只接收当前 `triggerTurnId` 对应的 suggest 返回，过期请求结果会被丢弃。

```typescript
// 当前 suggest 主链路伪代码（以仓库实现为准）
serve(async (req) => {
  const llm = createLlmRuntime(); // provider/model 由环境变量决定
  const responseHeaders = buildLlmResponseHeaders(llm, {
    "Content-Type": "application/json",
  });

  const recentTurns = await loadRecentTurns(sessionId);
  const prompt = buildQuickReplyPrompt(recentTurns, lastUtterance, scene);

  const completion = await llm.client.chat.completions.create(
    withLlmDefaults(llm, {
      messages: prompt,
      max_tokens: 100,
      temperature: 0.4,
    }),
  );

  const cleanText = sanitizeSuggestionText(
    completion.choices[0]?.message?.content ?? "",
  );

  return new Response(
    JSON.stringify({
      suggestions: cleanText ? [{ style: "simple", text: cleanText }] : [],
    }),
    { headers: responseHeaders },
  );
});
```

```typescript
class SuggestionService {
  async fetchSuggestions(
    sessionId: string,
    lastUtterance: string,
```

## 五、模块三：即时语言 Review

### 5.1 Edge Function：Review

**当前实现对齐**：

- `review` 仍为非流式 JSON。
- 短句跳过逻辑保留。
- `review` 与 `suggest` 共用 `_shared/llm.ts`，目前支持通过环境变量切换：
  - `openai`
  - `gemini`
  - `groq`
  - `deepseek`
  - `minimax`
- 默认可按 `LLM_PROVIDER` / `LLM_MODEL` 切换，且响应头会返回 provider/model/key prefix 便于调试。

```typescript
// 当前 review 主链路伪代码（以仓库实现为准）
serve(async (req) => {
  const llm = createLlmRuntime();
  const responseHeaders = buildLlmResponseHeaders(llm, {
    "Content-Type": "application/json",
  });

  if (tooShort(userUtterance)) {
    return new Response(JSON.stringify(skippedReview), { headers: responseHeaders });
  }

  const recentTurns = await loadRecentTurns(sessionId, 6);
  const completion = await llm.client.chat.completions.create(
    withLlmDefaults(llm, {
      messages: buildReviewPrompt(recentTurns, userUtterance, scene),
      max_tokens: 200,
      temperature: 0.3,
      response_format: { type: "json_object" },
    }),
  );

  const review = JSON.parse(completion.choices[0].message.content ?? "{}");
  await persistReviewAsync(review);
  return new Response(JSON.stringify(review), { headers: responseHeaders });
});
```

### 5.2 客户端 Review 集成

**当前实现对齐**：

- 一轮 turn 的真实提交顺序已调整为：
  1. Deepgram 在同一 utterance 内累计 final 片段
  2. `UtteranceEnd` 到来后先快照当前 turn
  3. 先写入 `turns`
  4. 再触发 `suggest` 或 `review`
- `recordTurn()` 内部带 3 次指数退避重试，降低与下游请求并发时的瞬时网络失败概率。
- 本人一句话结束时，`suggest` 会在进入 review 链路前立即隐藏，而不是等 review 完成才消失。

```typescript
// 在 AudioEngine 的 onUtteranceEnd 回调中
async function handleUtteranceEnd(speaker: string, text: string) {
  // 1. 存储这轮对话
  await supabase.from( turns ).insert({
    session_id: currentSessionId,
    speaker,
    text,
    turn_id: `turn_${Date.now()}`,
  });

  if (speaker ===  other ) {
    // 2a. 对方说完 → 触发建议生成
    suggestionService.fetchSuggestions(currentSessionId, text, currentScene);
  } else if (speaker ===  self ) {
    // 2b. 自己说完 → 触发 Review
    const review = await fetch(`${SUPABASE_URL}/functions/v1/review`, {
      method:  POST ,
      headers: {
         Authorization : `Bearer ${accessToken}`,
         Content-Type :  application/json
      },
      body: JSON.stringify({
        session_id: currentSessionId,
        user_utterance: text,
        scene: currentScene,
      }),
    }).then(r => r.json());

    // 3. 展示 Review 结果
    useReviewStore.getState().setReview(review);
  }
}
```

### 6.2 用量控制（Free 用户每天 15 分钟）

```sql
-- Edge Function 中调用此函数检查用量
create or replace function check_daily_usage(p_user_id uuid)
returns json as $$
declare
  v_profile profiles%rowtype;
  v_limit integer;
begin
  select * into v_profile from profiles where id = p_user_id;

  -- 日期翻转则重置计数
  if v_profile.daily_minutes_reset_at < current_date then
    update profiles
    set daily_minutes_used = 0, daily_minutes_reset_at = current_date
    where id = p_user_id;
    v_profile.daily_minutes_used := 0;
  end if;

  -- Free 用户 15 分钟，Pro 不限
  v_limit := case v_profile.subscription_tier
    when 'free' then 15
    else 999999
  end;

  return json_build_object(
    'used', v_profile.daily_minutes_used,
    'limit', v_limit,
    'exceeded', v_profile.daily_minutes_used >= v_limit
  );
end;
$$ language plpgsql security definer;
```
