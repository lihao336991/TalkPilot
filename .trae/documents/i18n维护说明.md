# TalkPilot i18n 维护说明

## 1. 当前目标

本项目已落地第一版本地 i18n 基础设施，当前覆盖范围：

- 根布局初始化与语言切换
- 本地语言设置持久化
- `Profile`
- `Paywall`
- `CustomerCenter`
- `Settings`
- `Login / AuthActionPanel`
- `Onboarding`
- `CustomTabBar / tabs 标题 / Not Found`
- `History`
- `Coach`
- `Live` 高频组件
  - `NativeAssistCard`
  - `StartSessionCard`
  - `ConversationToolbar`
  - `ConversationFlow`
  - `TranscriptBubble`
  - `ReviewIndicator`
  - `ReviewDetailCard`
  - `SuggestionPanel`
  - `SuggestionCard`
  - `VoiceEnrollmentCard`
  - `SpeakerCalibration`

当前首批支持语言：

- `en`
- `zh-CN`

后续扩语言时，优先沿用这套结构继续扩展，不要再回到页面里直接写死文案。

## 3. 各文件职责

- `src/shared/i18n/config.ts`
  - 定义支持语言
  - 处理系统语言归一化
  - 提供 `getSystemUiLocale()`

- `src/shared/i18n/init.ts`
  - 初始化 `i18next`
  - 注册本地资源

- `src/shared/i18n/types.ts`
  - 基于英文资源生成翻译结构类型
  - 用于约束其他语言包必须同构

- `src/shared/i18n/locales/en.ts`
  - 英文基准语言
  - 所有新 key 必须先加这里

- `src/shared/i18n/locales/zh-CN.ts`
  - 中文资源

- `src/shared/i18n/useAppLanguage.ts`
  - 提供 `useAppLanguage()`
  - 提供 `useI18nBootstrap()`

- `src/shared/store/localeStore.ts`
  - 持久化语言偏好
  - 当前保存：
    - `followSystemUiLocale`
    - `uiLocale`
    - `helpLocale`
    - `targetLanguage`

## 4. 当前语言模型

第一版虽然只接入了 UI 语言，但数据模型已经按后续多语学习产品预留为三层：

- `uiLocale`
  - 控制 App 静态界面文案
- `helpLocale`
  - 预留给 AI 解释语言
- `targetLanguage`
  - 预留给学习目标语言

当前实际生效的是：

- `uiLocale`
- `targetLanguage`

当前已持久化但尚未在业务里大规模消费的是：

- `helpLocale`

当前设置入口：

- `Profile` 页内进入 `settings`
- 设置页当前提供两组偏好：
  - `App language`
  - `Learning language`

说明：

- `App language` 已真实接管界面文案
- `Learning language` 当前已真实持久化，并已接入主/assist websocket、review、suggest、assist-reply 的语言参数
- 当前生成链路语言规则已收敛为：
  - `suggest` 输出跟随 `Learning language`
  - `review` 的解释/表扬输出跟随 `App language`
  - `review` 的修正句与更自然表达仍保留 `Learning language`
- 当前产品语言模型已进一步收敛为：
  - `uiLocale`：当前承载母语文案
  - `learningLanguage`：当前承载学习语言
- 详细链路说明见 `.trae/documents/双语言模型说明.md`

## 5. 页面里怎么用

统一使用：

```ts
import { useTranslation } from "react-i18next";
```

或后续统一收口到：

```ts
import { useAppLanguage } from "@/shared/i18n";
```

页面内写法示例：

```ts
const { t } = useTranslation();

<Text>{t("profile.headerTitle")}</Text>
```

带插值示例：

```ts
t("billing.paywall.continueForPrice", {
  price: selectedPackage.product.priceString,
});
```

## 6. 新增文案的硬规则

新增任何用户可见文案时，必须遵循下面顺序：

1. 先在 `src/shared/i18n/locales/en.ts` 新增 key
2. 再在 `src/shared/i18n/locales/zh-CN.ts` 补齐对应翻译
3. 页面里改成 `t("...")`
4. 不要在组件里长期保留裸字符串

不要这样做：

```ts
<Text>Purchase complete</Text>
```

应该这样做：

```ts
<Text>{t("billing.paywall.purchaseCompleteTitle")}</Text>
```

## 7. key 命名规范

当前统一采用：

- `common.*`
- `profile.*`
- `live.*`
- `billing.paywall.*`

命名要求：

- key 表达语义，不表达英文句子本身
- 不使用 `title1`、`text2` 这类弱语义命名
- 能复用的动作和状态优先放 `common`

推荐：

- `common.actions.retry`
- `profile.signOutConfirmTitle`
- `billing.paywall.package.savePercent`

不推荐：

- `retryButtonText`
- `profileText1`
- `saveText`

## 8. 什么时候放进 i18n

应该放进 i18n 的：

- 按钮
- 标题
- Alert 文案
- 错误提示
- 空态
- 状态标签
- 购买与恢复文案
- 固定教学模板

不要放进 i18n 的：

- AI 实时生成的解释内容
- AI 生成的 review / suggest 正文
- 用户输入
- 动态服务端返回内容

## 9. TalkPilot 的边界约定

针对当前产品，建议长期遵守：

- `UI 文案` 走本地 i18n
- `AI 解释语言` 未来由 `helpLocale` 控制
- `学习目标语言` 未来由 `targetLanguage` 控制

也就是说，不要把所有语言逻辑都塞进一个单独 `locale` 字段里。

## 10. AI 维护推荐工作流

推荐使用下面这套流程让 AI 帮忙维护文案：

1. 开发者只改页面和 `en.ts`
2. 让 AI 对照 `en.ts` 补 `zh-CN.ts`
3. 人工 review 高风险文案
4. 合并后再继续扩更多语言

给 AI 的指令建议固定成这种格式：

```txt
请根据 src/shared/i18n/locales/en.ts 中新增的 key，
补齐 src/shared/i18n/locales/zh-CN.ts。

要求：
1. 只补缺失 key，不改已有 key
2. 结构必须和 en.ts 完全一致
3. TalkPilot 不翻译
4. Pro 默认保留英文
5. 移动端文案要短、自然、可读
6. 插值变量如 {{price}} {{language}} 必须保留原样
```

## 11. 扩语言步骤

未来新增 `ja`、`ko`、`es` 等语言时：

1. 新建 `src/shared/i18n/locales/ja.ts`
2. `import type { TranslationSchema } from "../types"`
3. 导出 `const ja: TranslationSchema = { ... }`
4. 在 `init.ts` 里注册资源
5. 在 `config.ts` 的 `SUPPORTED_UI_LOCALES` 中加入语言 code

## 12. 推荐的下一步

建议按下面顺序继续扩：

1. 迁移 `app/login.tsx`
2. 迁移 `LiveScreen` 和相关卡片
3. 迁移 `History` / `Coach`

## 13. 当前已知限制

- 当前资源文件还是“每种语言一个总文件”
  - 语言和页面继续增多后，可以拆成按业务域分文件
- 当前没有自动化脚本校验 missing keys
  - 后续建议补：
    - `i18n:check`
    - `i18n:report`
    - `i18n:raw-string`
- 当前 `helpLocale` / `targetLanguage` 只是预留，尚未接入后端 prompt
