# TalkPillot — 实时英语对话副驾驶 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品名称

**TalkPillot** — Real-time English Conversation Co-pilot

### 1.2 产品定位

TalkPillot 不是一款传统的英语学习 App，而是一款 **实时英语对话副驾驶**。它在用户真实的英语交流场景中常驻运行，通过实时监听对话、智能生成回复建议、即时语法纠正三大核心能力，帮助用户在真实场景中边用边学，大幅降低英语沟通的心理门槛和表达障碍。

### 1.3 产品愿景

> **让每一次真实的英语对话，都成为一次高质量的学习机会。**

### 1.4 核心差异化

| 维度         | 传统语言学习 App  | TalkPillot                   |
| ------------ | ----------------- | ---------------------------- |
| 学习场景     | 虚拟预设课程/任务 | 用户真实生活、工作、留学场景 |
| 学习方式     | 刷题式、任务驱动  | 实时对话参与、即时生成+纠正  |
| 内容来源     | 平台编排的教材    | 用户当下真实的对话内容       |
| 使用频率     | 每天打卡 15 分钟  | 常驻型，说英语时就打开       |
| 核心价值     | 知识输入          | 实战输出 + 即时反馈          |
| 用户粘性机制 | 游戏化激励/打卡   | 真实需求场景天然绑定         |

### 1.5 一句话描述

## **TalkPillot = Otter.ai 的实时转录能力 + Grammarly 的语言纠错能力 + 实时 AI 回复建议，融合在用户真实对话场景中。**

## 三、核心功能设计

### 3.1 功能架构总览

_(diagram)_

### 3.2 P0 功能（MVP 必须交付）

#### 3.2.1 实时对话引擎

**功能描述**：App 开启后，实时监听并识别用户所处的英语对话，将双方语音转为文字，并区分说话人身份。
**核心流程**：

1. 用户点击 开始对话 按钮，App 开始监听
2. 系统通过流式 ASR 将语音实时转为文字
3. 通过说话人分离技术（Speaker Diarization）区分 对方 和 我
4. 识别结果实时展示在屏幕上，形成滚动对话流
   **说话人区分方案**：

| 方案                    | 原理                                               | 优势               | 劣势                             | 推荐度           |
| ----------------------- | -------------------------------------------------- | ------------------ | -------------------------------- | ---------------- |
| 方案 A：耳机+手机双通道 | 耳机麦克风采集自己的声音，手机麦克风采集对方的声音 | 物理隔离，准确度高 | 依赖耳机，场景受限               | 推荐（MVP 首选） |
| 方案 B：声纹识别        | 用户首次使用时录入声纹，系统自动区分               | 无需额外设备       | 首次设置成本，嘈杂环境准确度下降 | 备选             |
| 方案 C：手动标记        | 用户按住按钮标记 对方在说                          | 实现简单           | 交互负担重，影响对话自然度       | 不推荐           |

**技术规格**：

- ASR 延迟要求：< 800ms
- 说话人分离准确率目标：> 90%
- 支持语言：英语（美式/英式/澳式口音）
- 环境适应：需支持中等噪音环境（咖啡厅、教室）

#### 3.2.2 AI 实时回复建议（核心差异化功能）

**功能描述**：当对方说完一句话后，AI 基于对话上下文，在 1-3 秒内生成 2-3 条英文回复建议，展示给用户参考。用户可以跟读建议，也可以自由发挥。
**核心流程**：

1. 对方说完一句话（系统检测到对方语音停顿 > 1.5 秒）
2. AI 结合对话上下文 + 场景预设，生成 2-3 条回复建议
3. 建议以卡片形式浮现在屏幕上，按不同风格排列
4. 用户可以：

- **跟读**：点击建议，系统朗读一遍，用户跟读
- **自由发挥**：直接用自己的话回复，系统进入 Review 模式
- **忽略**：继续听对方说话
  **回复建议风格**：

| 风格标签 | 说明         | 示例（对方说 Would you like to grab coffee? ）                |
| -------- | ------------ | ------------------------------------------------------------- |
| Formal   | 正式场合用语 | That sounds great. I'd love to, thank you for the invitation. |
| Casual   | 日常口语     | Sure, I'm down! Where do you wanna go?                        |
| Simple   | 简洁表达     | Yeah, sounds good!                                            |

**关键交互设计**：

- 建议卡片需支持左滑切换风格
- 建议文字应同时展示音标/重音标记（可选开关）
- 支持点击单词查看释义
  **技术规格**：
- 生成延迟要求：对方停顿后 < 2 秒展示第一条建议
- 建议数量：默认 2 条，可设置 1-3 条
- 上下文窗口：最近 10 轮对话

#### 3.2.3 即时语言 Review

**功能描述**：用户自由发挥说完一句话后，AI 即时对用户的表达进行语法、用词、表达地道性的评估，并给出轻量级反馈。
**核心流程**：

1. 用户说完一句话（系统检测到用户语音停顿 > 2 秒）
2. AI 对用户这句话进行多维度评估
3. 轻量反馈即时展示（不打断对话节奏）
4. 详细纠正内容存入对话复盘
   **评估维度**：

| 维度       | 评估内容                   | 反馈形式                  |
| ---------- | -------------------------- | ------------------------- |
| 语法正确性 | 时态、主谓一致、介词搭配等 | 错误词标红 + 正确形式浮窗 |
| 用词地道性 | 是否使用母语者常用表达     | 建议更地道的替代词/短语   |
| 表达完整性 | 是否完整表达了意思         | 建议补充内容              |
| 综合评分   | 本句话整体质量             | 颜色指示（绿/黄/红）      |

**轻量反馈设计原则**：

- 对话进行中：只展示颜色指示条（绿=优秀，黄=可以更好，红=有明显错误）
- 用户点击颜色条：展开详细纠正卡片
- 所有详细内容自动存入对话复盘

### 3.3 P1 功能（快速跟进）

#### 3.3.1 场景预设与管理

**功能描述**：用户可以在开始对话前预设当前场景，AI 会根据场景调整回复建议的风格和专业度。
**预设场景模板**：

| 场景分类 | 具体场景                                  | AI 调整策略          |
| -------- | ----------------------------------------- | -------------------- |
| 校园学术 | 课堂讨论、Office Hour、学术演讲、论文答辩 | 偏学术用语，更正式   |
| 日常生活 | 租房看房、超市购物、餐厅点餐、看病就医    | 偏日常口语，简洁实用 |
| 职场工作 | 工作会议、同事闲聊、绩效面谈、商务谈判    | 偏职场用语，专业得体 |
| 社交娱乐 | 朋友聚会、约会、运动社交                  | 偏轻松口语，地道俚语 |
| 自定义   | 用户自行描述场景                          | AI 根据描述动态调整  |

- 支持用户创建自定义场景并保存
- 也支持不预设，进入 自由对话模式 ，AI 根据对话内容自动判断场景

#### 3.3.2 对话复盘报告

**功能描述**：每次对话结束后，生成结构化的复盘报告。
**报告内容**：

- 对话全文（双方完整对话记录，带时间戳）
- 错误汇总（所有被标记的语法/用词问题，附正确形式）
- 新词收录（对话中出现的用户可能不熟悉的词汇/表达）
- 本次亮点（用户表达得好的句子，正向反馈）
- 对话能力评分（流利度、准确度、词汇丰富度、响应速度四维雷达图）

| 高频场景句型库   | 从用户真实对话中沉淀高频表达，按场景分类，支持复习 |

## 五、交互设计要点

### 5.1 主界面布局

主界面采用 **对话流 + 底部建议栏** 的双层布局：

- **上半部分**：滚动对话流，区分对方（左侧灰色气泡）和我（右侧蓝色气泡），实时更新
- **下半部分**：AI 建议区域，对方说完后自动浮现建议卡片
- **底部**：操作栏（暂停/继续、场景切换、结束对话）

### 5.2 Review 反馈交互

对话进行中的反馈必须 **极轻量**，设计原则：

| 层级        | 触发方式         | 展示内容                       | 时长           |
| ----------- | ---------------- | ------------------------------ | -------------- |
| L1 即时反馈 | 用户说完自动出现 | 句子下方颜色指示条（绿/黄/红） | 常驻直到下一句 |
| L2 快速详情 | 用户点击颜色条   | 错误标注 + 正确形式弹窗        | 用户主动关闭   |
| L3 完整复盘 | 对话结束后自动   | 完整复盘报告页面               | 永久保存       |

### 5.3 回复建议展示

- 建议卡片从底部滑入，半透明背景，不遮挡对话流
- 支持左右滑动切换不同风格（Formal / Casual / Simple）
- 点击卡片可触发 TTS 朗读，用户跟读
- 如果用户开始自己说话，建议卡片自动收起并淡出

## 六、隐私与合规

### 6.1 核心隐私策略

| 策略       | 说明                                                                   |
| ---------- | ---------------------------------------------------------------------- |
| 知情同意   | App 首次使用时明确告知录音功能，需用户确认                             |
| 对方知情权 | 建议用户在使用前告知对话对方正在使用语言辅助工具（非强制）             |
| 数据存储   | 语音数据仅在端侧暂存用于 ASR，不上传保存原始音频；文字对话记录加密存储 |
| 数据删除   | 用户可随时删除所有对话记录                                             |
| 第三方合规 | 遵循 GDPR（欧洲）、CCPA（加州）相关规定                                |

### 6.2 各地区录音法规适配

| 地区             | 法规要求                      | App 策略                   |
| ---------------- | ----------------------------- | -------------------------- |
| 美国（多数州）   | 单方同意（One-party consent） | 默认开启，提示用户         |
| 加州、伊利诺伊等 | 双方同意（Two-party consent） | 提供 对方已知情 确认按钮   |
| 欧盟             | GDPR 严格隐私保护             | 仅处理语音不存储，端侧优先 |
| 英国             | 类 GDPR                       | 同上                       |
| 澳大利亚         | 各州不同                      | 默认保守策略               |

## 十、关键风险与应对

| 风险             | 影响                         | 应对策略                                           |
| ---------------- | ---------------------------- | -------------------------------------------------- |
| 实时延迟超标     | 建议出现太慢，对话已经过去   | 端侧 ASR 预处理 + 轻量 LLM 优先 + 预测性生成       |
| 说话人分离不准   | AI 把对方的话当成用户的话    | MVP 先用耳机方案物理隔离，后续迭代声纹方案         |
| 嘈杂环境识别率低 | 转录错误导致建议不相关       | 接入降噪模型 + 展示置信度 + 用户可手动纠正         |
| 隐私合规风险     | 录音法规不同可能导致法律风险 | 上线前逐市场法务审核，保守策略优先                 |
| LLM API 成本     | 高频实时调用导致成本过高     | 用量控制（Free 版限时）+ 小模型优先 + 缓存常见回复 |
| 用户习惯培养     | 用户不习惯对话时开着 App     | 首次使用引导 + 场景化推送提醒 + 对话后成就感反馈   |

## 十一、成功指标

### 11.1 North Star Metric

**周活跃对话时长**（Weekly Active Conversation Minutes）——用户每周在真实对话场景中使用 TalkPillot 的总时长。

### 11.2 核心指标体系

| 指标类别 | 具体指标                    | MVP 目标         |
| -------- | --------------------------- | ---------------- |
| 获客     | 下载量                      | 上线首月 10,000+ |
| 激活     | 首次完成完整对话的用户比例  | > 60%            |
| 留存     | 7 日留存率                  | > 40%            |
| 参与度   | 平均每次对话使用时长        | > 8 分钟         |
| 参与度   | 回复建议采纳率（跟读/参考） | > 30%            |
| 付费     | 免费转付费转化率            | > 5%             |
| 满意度   | NPS 评分                    | > 40             |
