# 🧾 一、PRD（产品需求文档）

## 1. 产品目标

打造一个**实时对话辅助能力**：

- 实时识别对话内容（ASR）
- 实时判断：
  - **哪句话是用户说的**
  - **哪句话是对方说的**

- 为后续功能服务：
  - 翻译
  - AI建议（reply suggestion）
  - 语法纠错

👉 核心能力：

> **稳定区分「我 vs 对方」**

---

## 2. 用户场景

### 🎯 场景1：面对面对话（主场景）

- 用户与1~2人对话
- 手机放桌上或手持
- App后台实时监听

---

### 🎯 场景2：电话 / 语音交流（扩展）

- 用户戴耳机
- 自己声音 + 外放声音

---

## 3. 核心功能

### 3.1 声纹注册（一次性）

用户首次使用：

- 引导录音：
  - 3~5段语音
  - 每段 2~5秒

- 系统生成：
  - 用户声纹 embedding（向量）

👉 输出：

- user_voice_embedding（本地存储）

---

### 3.2 实时语音识别

使用 Deepgram：

- 实时 ASR
- 返回：
  - 文本
  - 时间戳

---

### 3.3 说话人归属判断（核心能力）

对每段语音做：

👉 判断：

- 是否为“用户本人”

输出：

```text
{
  text: "hello how are you",
  speaker: "user" | "other"
}
```

---

### 3.4 对话流展示（核心 UI 逻辑）

统一流式展示：

```text
[对方] Hello, how are you?
[我] I'm good, thanks.
[AI建议] You can say: "I'm doing great!"
```

---

## 4. 非功能需求

### ⚡ 实时性

- 延迟 < 300ms（识别+判断）

### 📱 端侧运行

- 声纹识别必须本地完成
- 不依赖网络

### 🔋 性能

- CPU占用可控
- 不明显耗电

---

## 5. 成功指标（KPI）

- 用户识别准确率 > 90%
- speaker抖动率 < 5%
- 用户无感知错误

# 🏗️ 二、技术架构设计

---

## 🧠 1. 总体架构

```text
音频输入
   ↓
VAD（语音检测）
   ↓
分段（chunk）
   ↓
├── Deepgram（ASR）
│      ↓
│   文本流
│
└── Resemblyzer（端侧）
       ↓
   speaker embedding
       ↓
   similarity判断（我/非我）

最终融合
   ↓
对话流输出
```

---

## 🧩 2. 核心模块拆解

---

### 2.1 音频输入

- iOS：AVAudioEngine
- Android：AudioRecord

采样：

- 16kHz
- mono

---

### 2.2 VAD（语音活动检测）

推荐：

- WebRTC VAD（轻量稳定）

作用：

👉 只在“有说话”时触发后续计算

---

### 2.3 音频分段策略（关键）

- 每段：
  - 1.5 ~ 2秒

- 滑动窗口：
  - 0.5秒步长

👉 保证：

- 足够信息
- 实时性

---

### 2.4 Resemblyzer 推理

使用：

Resemblyzer

处理流程：

```text
音频段 → embedding（256维向量）
```

---

### 2.5 声纹比对

方法：

👉 cosine similarity

```text
similarity = cos(user_embedding, current_embedding)
```

---

### 2.6 判定策略（非常关键）

#### 阈值判断

```text
> 0.75 → 用户
< 0.65 → 非用户
中间 → 不确定
```

---

#### 多帧投票

```text
连续3帧判定为用户 → 才输出 user
```

---

#### 状态机（防抖）

```text
当前是 user：
  若下一帧略低 → 仍保持 user

当前是 other：
  需连续高置信度 → 才切换 user
```

---

### 2.7 动态声纹更新（进阶）

当：

```text
similarity > 0.85
```

👉 更新用户 embedding：

```text
user_embedding = 0.9 * old + 0.1 * new
```

---

### 2.8 Deepgram ASR

使用：

Deepgram

获取：

- 文本
- 时间戳

---

### 2.9 数据融合（关键）

按时间轴对齐：

```text
ASR片段 ↔ speaker判定
```

输出统一结构：

```json
{
  "text": "...",
  "speaker": "user",
  "start": 123,
  "end": 456
}
```
